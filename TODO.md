## Open questions

- Which Japanese-focused disc database is best as the first JP-region provider? Candidates: VGMdb, Tower Records Japan, CDJapan, HMV Japan. Needs research into which has a usable API.
- **Sprint 13 `list --filter` vocabulary on `genre` / `language`.** Resolved Sprint 13: MVP ships `--artist`, `--year`, `--country`, `--label` (all schema-backed on `Album` / `Release`) and drops `genre` / `language`. Reopening them requires two pieces: a schema column or associative table, and a provider enrichment pass (MB exposes genre via release-group tags and language via work-level MBID links — neither fits as a simple column). When both land, the flags slot into the existing `ListFilters` struct at `phono-junk-lib/src/list.rs`.
- How to model HDCD / SACD / mixed-mode carriers — extend `Disc` with a carrier-type enum, or add to `extra` map?
- User-facing vs. archival override semantics — should overrides fully replace scraped data, or sit alongside it as "preferred display" while keeping the provider's value for audit?
